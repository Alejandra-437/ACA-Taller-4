    Acerca de este manual#

    Sobre el proyecto#

    Este manual fue creado como parte del Taller de MkDocs del curso de Aplicación De Código Abierto (ACA).

    El objetivo es demostrar las capacidades de MkDocs con el tema Material for MkDocs para crear documentación técnica de alta calidad.

    Objetivos de aprendizaje#

    Al completar este taller, se cubren los siguientes objetivos:

    Objetivo Estado
    Crear un proyecto MkDocs desde cero
    Configurar el tema Material for MkDocs
    Personalizar paleta de colores y tipografía
    Crear múltiples páginas con navegación
    Usar extensiones de Markdown (admoniciones, tabs, etc.)
    Configurar plugins (search, git-revision-date)
    Incluir imágenes, tablas y bloques de código
    Publicar en GitHub Pages

    Tecnologías utilizadas#

    • Python 3.13.0 - Lenguaje de programación
    • MkDocs 1.6.1 - Generador de sitios estáticos
    • Material for MkDocs - Tema visual
    • PyMdown Extensions - Extensiones de Markdown
    • Git - Control de versiones
    • GitHub Pages - Hosting del sitio

    Requisitos cubiertos#

    Este manual cumple con todos los requisitos del taller:

    • Tabla: Presente en cada página principal.
    • Bloque de código con pestañas (tabs): Múltiples ejemplos con === "Lenguaje".
    • Enlace interno entre páginas: Navegación completa con nav:.
    • Imagen almacenada en docs/img/: Logo de Git en la página principal.
    • Bloque de admonición: Notas, tips, warnings en cada página.

    Cómo contribuir#

    Si encuentras errores o quieres mejorar este manual:

    1. Haz un fork del repositorio
    2. Crea una rama con tu cambio (git checkout -b mejora/seccion)
    3. Realiza los cambios y haz commit
    4. Sube la rama y abre un Pull Request
